Stack and queue栈与队列

Stack栈

栈的概念

栈是一种先进后出(FILO)的数据结构。可以将其理解为一个一端封闭的羽毛球筒，想要取出更早放入的羽毛球，就要将更迟放入的羽毛球取出来。

eg.逆波兰表达式

考虑仅包含单字母变量、圆括号与+、-、*、/的算术式，给出中缀表达式，求出其逆波兰表达式（后缀表达式）。

深度优先搜索

搜索树

我们这样定义搜索树的结点与接电脑关系：

  1. 搜索树的一个结点表示在问题求解中的一个状态。
  2. 若一个状态能由另一个状态单步转移得到。则前者是后者的子结点。
  3. 搜索树的根节点的状态是问题的初始状态。

搜索，即是遍历搜索树的过程。

深度优先搜索，顾名思义，就是在遍历时，先遍历当前结点的子结点。当当前结点无可遍历子结点时，回到其父节点寻找可遍历的其兄弟结点。重复此过程直到搜索树被完全遍历或遍历被人工终止。

递归(recursion)与回溯(backtracking)

一个递归关系的构成包括递归基础，和递归步骤两部分。举个例子，一个人的祖先可以如此递归定义：一个人的双亲是他的祖先（递归基础），一个人的祖先的双亲同样是该人的祖先（递归步骤）。 可用上述方法遍历到的对象即为该人的祖先。不难发现这种调用自身的思想与深度优先搜索的实现需求不谋而合。在深度优先搜索中，我们从搜索树的根节点开始，搜索的方法即对以其子结点为根节点的搜索子树展开搜索。这个过程中可以反复的自身调用搜索方法直至找到答案或者完成搜索树的遍历。

回溯的过程即是指撤销当前操作，避免当前操作对下一种操作的影响。

eg.字符集的排列组合问题

给定一个字符集，求其所有可能的全排列和组合方案。

eg.八皇后问题

即在一个8*8的棋盘上放置八个皇后，求使它们无法互相攻击的方案。

深度优先搜索与手动建栈

不难发现，深度优先搜索与栈的关系很密切。我们往往采用的是下面的编程模板：

[type] [function name](arguments) {
    if (<son>.is_empty()) {
        // some operations
        return [value];
    }
    for ([choice] in <son>) {
        //makechoice([choice]);
        //recursion
        //unmakechoice([choice]);
    }
    return [value];
}

我们在不断调用自身进行深度优先搜索的过程中，即使将栈顶状态的子状态压入栈中，当栈顶状态不再拥有可遍历的子状态时，弹栈。当栈为空完成搜索。我们在直接使用上述模板实现的时候，调用的是系统栈。 但当问题规模比较大的时候，系统栈可能就无法负担我们的需求了（尤其在Windows环境下，Windows的系统栈比较小）。

一种解决方法是手工建栈。即自己实现这个在栈中反复调用自身的过程。

Queue队列

队列的概念

队列是一种先进先出(FIFO)的数据结构。就好像一个在饭堂打饭的队伍，越排在前面的人越早打到饭。队列的种类有顺序队列、循环队列等。

队列和栈都是线性表的一种，栈只允许在结尾进行插入和删除操作，队列只允许在开头删除，在结尾插入。对该部分有兴趣可再自行查找资料学习。

宽度优先搜索

宽度优先搜索，即在遍历完当前结点后，优先遍历其兄弟结点。若已无兄弟结点再遍历其子节点。结合队列的性质，不难发现用队列即可实现广度优先搜索。

eg.简单的迷宫问题

给定一个m*n的矩阵，矩阵被1和-1填充，保证左上角和右下角的元素为1，问是否能从左上角遍历至右下角？最短需要几步？遍历规则如下：每次移动只能沿行或者列移动一格，不可移动至元素值为-1的位置。
